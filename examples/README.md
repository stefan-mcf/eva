# Examples

These examples are synthetic fixtures for understanding EVA input and output shapes. They are not live Hermes profile state, operator memory, session data, credentials, or generated vault artifacts.

- `example-scan.json` shows a compact scan bundle shape.
- `example-proposal.json` shows a reviewable proposal payload.
- `example-brief.md` shows a concise operator brief.
- `minimal-profiles/` contains a fake profile tree that can be used in local smoke tests.

The `minimal-profiles/example-profile/memory.md` file is intentionally small and generic. It models an adapter input file name without containing real runtime memory.
